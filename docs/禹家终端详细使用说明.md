# 禹家终端（大禹）详细使用说明

> 适用项目：禹家智能家居终端  
> 运行设备：OpenHarmony 大禹开发板  
> 文档版本：2026-07-18

## 1. 产品说明

禹家终端是一套运行在大禹开发板上的本地智能家居控制中心，主要负责：

- 显示室内外温湿度、天气、设备状态和安全预警；
- 控制照明、空调、窗帘和智能门锁；
- 通过本地语音识别执行常用控制命令；
- 通过人脸和声纹验证保护敏感操作；
- 使用 USB 数据线接入新的设备；
- 管理多块设备的能力绑定；
- 在 MQTT/HTTP 和 SM4/AES 之间切换；
- 在大禹本机运行 MQTT Broker 和 HTTP Bridge。

本系统的家庭局域网通信不再依赖常开电脑。大禹本机提供 Broker，设备只需与大禹处于同一家庭网络即可使用。

## 2. 硬件与网络准备

### 2.1 硬件组成

- 大禹开发板一台；
- 大禹配套显示屏；
- 麦克风和扬声器；
- 一块或多块设备控制板；
- USB Type-C 数据线，用于首次接入和配网；
- 家庭无线路由器。

当前已联调的设备包括：

| 设备 | 主要能力 |
| --- | --- |
| 传感器控制器 | 温湿度、照明、门锁、人体感应等 |
| 空调控制器 | 空调控制、红外学习、安全预警；当前窗帘也由该控制器承载 |

### 2.2 网络要求

1. 大禹和所有设备必须连接同一个局域网。
2. 建议在路由器中为大禹设置 DHCP 地址保留，避免大禹 IP 变化。
3. 当前大禹局域网地址示例为 `192.168.3.103`，实际使用时以路由器分配地址为准。
4. 开启 TLS 时，设备使用以下地址：

```text
mqtts://大禹IP:8884
https://大禹IP:3444
```

5. 明文兼容端口仍然保留，但正式使用建议开启 TLS：

```text
mqtt://大禹IP:1883
http://大禹IP:3000
```

## 3. 启动与退出

### 3.1 启动

打开“禹家终端”应用后，系统会自动完成：

1. 启动本地 MQTT Broker；
2. 启动本地 HTTP/HTTPS Bridge；
3. 连接本机 Broker；
4. 等待设备上线并发送 `device_hello`；
5. 加载人脸、声纹和语音模型；
6. 恢复已保存的设备注册与能力绑定；
7. 将默认控制偏好设为 MQTT 和 SM4。

设备随后发送的 `device_hello` 会用于同步其真实在线状态、控制通道、加密方式和能力。

### 3.2 退出

点击左侧栏底部的“退出”按钮即可关闭应用。

退出后，大禹应用内嵌的 Broker 也会随应用停止。家庭设备需要重新打开禹家终端后才能继续连接。

## 4. 界面概览

左侧导航栏包含两个主要页面：

- **总览**：显示时间、天气、环境数据和六个常用控制卡片；
- **设置**：管理设备、接入、红外、声纹、绑定、协议、加密和调试信息。

## 5. 总览页面

### 5.1 时间、日期与天气

页面左上区域显示：

- 当前时间；
- 日期和星期；
- 南京市江宁区天气；
- 室内温度；
- 室内湿度；
- 室外温度；
- 室外湿度。

室内数据来自声明 `sensor` 能力的设备，不受“设备状态”中当前选中设备影响。室外数据来自天气服务。

### 5.2 智能门禁

- 单击门锁卡片：在开锁和上锁之间切换；
- 双击门锁卡片：打开人脸管理界面；
- “正在同步门锁”表示命令已经发出，正在等待设备 ACK；
- 收到匹配的 `deviceId` 和 `seq` 后，界面才会更新最终状态。

门锁命令自动发送给声明 `door` 能力的设备。即使当前选中的是空调控制器，门锁仍会发送到门锁绑定设备。

### 5.3 照明控制

- 单击照明卡片即可开灯或关灯；
- 灯泡图标会随真实灯光状态变亮或变暗；
- 命令发送后显示同步状态；
- 收到设备 ACK 后更新为“灯已开启”或“灯已关闭”。

照明命令自动发送给绑定了 `light` 能力的设备，不依赖当前选中设备。

### 5.4 空调控制

单击空调卡片打开控制弹窗，可进行：

- 开机、关机；
- 温度升高、降低；
- 制冷、制热切换；
- 品牌/控制方案选择（如界面提供）；
- 查看当前温度、模式和执行状态。

空调命令自动发送到绑定了 `ac` 能力的设备。

### 5.5 窗帘控制

在窗帘图标上左右滑动即可调整开合度：

- `0%` 表示完全关闭；
- `100%` 表示完全打开；
- 松手后发送最终开合度；
- 收到设备 ACK 后同步显示真实百分比。

窗帘使用独立的 `curtain` 能力绑定。当前窗帘与空调位于同一块控制器时，系统会自动回退到空调绑定设备，不需要先选中该设备。

### 5.6 语音助手

语音助手无需点击按钮。当前唤醒词为：

```text
小禹小禹
```

建议使用方式：

1. 面向大禹正常音量说“小禹小禹”；
2. 听到唤醒提示后说出命令；
3. 等待语音播报执行结果。

当前支持本地语义解析的操作：

- 开灯；
- 关灯；
- 开空调；
- 关空调。

不要求使用固定完整句式。例如“把客厅灯打开”“打开照明”“帮我关掉空调”等近义表达可以被解析，但包含否定、询问或不支持房间的句子不会直接执行。

开门、解锁等高风险语音不会直接执行。

### 5.7 添加设备

单击“添加设备”卡片，会打开与设置页“设备接入”相同的 USB 接入界面。详细步骤见第 7 节。

## 6. 安全预警

当设备上报烟雾、水浸等一级预警时：

1. 总览页天气位置下方出现红色预警文字；
2. 右侧六个控制卡片区域切换为浅红色报警主题；
3. 大禹播放或显示报警提示；
4. 点击“取消报警”小按钮；
5. 确认后，大禹将消警命令发送回最初上报警情的设备；
6. 收到该设备 ACK 后，提示变为绿色“预警已解除，设备确认完成”。

消警命令根据报警来源设备自动路由，不受当前选中设备影响。

## 7. 接入新设备

### 7.1 接入前准备

1. 使用支持数据传输的 USB Type-C 线连接设备和大禹；
2. 确保设备已上电；
3. 打开“设置 → 设备接入”，或在总览页点击“添加设备”；
4. 建议保持“设备编号自动生成”；
5. 正式使用建议打开 TLS。

### 7.2 标准接入流程

按以下顺序操作：

1. 点击“扫描”，选择正确的 USB 设备；
2. 点击“自动探测”；
3. 等待界面显示设备已进入配网模式；
4. 选择家庭 Wi-Fi，并填写 Wi-Fi 密码；
5. 保持设备编号为自动生成；
6. 根据需要启用 TLS；
7. 点击“发送配置”；
8. 等待收到配置结果；
9. 点击“保存重启”；
10. 拔掉 USB 后等待设备通过 Wi-Fi 上线。

设备编号设为 `auto` 时，设备应使用 Base MAC 生成唯一 ID：

```text
esp32-XXXXXXXXXXXX
```

不要给多块设备配置相同的 `deviceId`。

### 7.3 接入成功的判断

设备重启上线后，应向大禹发送加密的 `device_hello`。大禹收到后会：

- 自动注册设备；
- 显示设备名称和编号；
- 记录设备能力；
- 记录 MQTT/HTTP 状态；
- 记录 SM4/AES 和 ECDH epoch；
- 自动绑定尚未分配的控制能力。

示例：

```json
{
  "type": "device_hello",
  "deviceId": "esp32-744DBD8A253C",
  "name": "Sensor Controller",
  "room": "",
  "capabilities": ["sensor", "door", "light"],
  "transport": "mqtt",
  "crypto": "sm4",
  "epoch": 1
}
```

## 8. 设置页面

设置页共有八个入口，按从左到右、从上到下排列。

### 8.1 设备状态

点击“设备状态”可查看：

- 已发现设备数量；
- 设备名称和唯一编号；
- 声明能力；
- 最近上线时间；
- 当前协议；
- 当前加密算法；
- 动态密钥状态；
- 当前控制目标。

点击列表中的设备可选择“协议和加密设置目标”。这个选择主要影响协议切换、加密切换和调试视图，不影响已经按能力绑定的照明、空调、窗帘、门锁控制。

### 8.2 设备接入

用于通过 USB 接入新设备。操作方法见第 7 节。

### 8.3 空调万能适配

该功能用于学习任意品牌空调遥控器的红外编码。

学习顺序为：

1. 开机；
2. 关机；
3. 温度加；
4. 温度减；
5. 制冷；
6. 制热。

操作步骤：

1. 点击“空调万能适配”；
2. 将遥控器对准设备上的红外模块；
3. 启动学习；
4. 根据屏幕提示按下当前按键；
5. 每个按键学习完成后等待系统自动进入下一项；
6. 六项完成后测试开机和关机；
7. 点击“完成并退出”。

注意事项：

- 大禹只发送一次 `learn-all`，不会同时发送单键 `learn`；
- 每个按键等待窗口最长约 65 秒；
- 完整流程可能需要约 6 分钟；
- 学习期间不要切换 MQTT/HTTP、SM4/AES，也不要手动触发 ECDH 重协商；
- 红外编码保存在红外模块自身 Flash 中，断电后不会丢失；
- 更换空调型号或遥控器后需要重新学习。

### 8.4 声纹保护

声纹验证在本地完成，不上传原始录音。

录入步骤：

1. 点击“声纹保护”；
2. 输入姓名或昵称；
3. 按提示完成三轮语音采集；
4. 每轮自然、连续地说出屏幕上的句子；
5. 三轮完成后，用户出现在授权列表；
6. 确认该用户的授权开关处于开启状态；
7. 开启声纹保护。

启用声纹保护后，语音命令执行前会验证说话人。验证失败时，大禹单独播放：

```text
声纹验证未通过，指令已拒绝
```

未授权指令不会发送给设备。

### 8.5 设备绑定

设备绑定用于指定每个控制卡片对应哪块设备。当前可绑定：

- 照明；
- 空调；
- 窗帘；
- 门锁。

设备上线时会根据 `capabilities` 自动绑定。需要手动调整时：

1. 点击“设备绑定”；
2. 在设备列表中找到目标设备；
3. 点击该设备下方的能力按钮；
4. 绿色按钮表示该能力当前绑定到此设备；
5. 灰色且不可点击表示设备没有声明该能力。

传感器、报警等上报能力会自动接入，不需要手动绑定控制卡片。

### 8.6 控制通道

支持两种设备控制通道：

- MQTT/MQTTS；
- HTTP/HTTPS。

切换前先在“设备状态”中选择目标设备，然后点击“控制通道”。系统会：

1. 使用设备当前通道发送 `proto` 切换命令；
2. 等待目标设备 ACK；
3. 收到 ACK 或目标通道确认包后更新图标；
4. 超时则保持原通道。

普通状态包只代表某条通道可达，不会覆盖已经确认的协议选择。

### 8.7 加密算法

支持：

- SM4-GCM；
- AES-GCM。

切换前先在“设备状态”中选择目标设备。每个设备分别维护：

- 当前加密算法；
- AES 动态密钥；
- SM4 动态密钥；
- ECDH epoch；
- 密钥协商状态。

切换一块设备不会直接修改其他设备的密钥或加密状态。

### 8.8 调试信息

连续点击“调试信息”图标三次可打开调试面板。调试信息包括：

- 消息时间；
- Topic；
- 来源设备；
- MQTT/HTTP 路由；
- ACK；
- 协议切换结果；
- 加密和解密状态。

普通用户无需长期打开该面板。联调时可在“当前设备”和“全部设备”之间切换。

## 9. 人脸识别与门锁

### 9.1 录入人脸

1. 双击总览页智能门禁卡片；
2. 输入姓名或昵称；
3. 保持面部位于相机画面中央；
4. 点击“人脸录入”；
5. 按提示完成五轮采集；
6. 录入完成后在授权用户列表中确认用户存在。

### 9.2 人脸识别控制

1. 打开人脸识别界面；
2. 点击“识别控制”；
3. 面向摄像头；
4. 识别为授权用户后，大禹根据当前门锁状态发送开锁或上锁命令；
5. 等待门锁设备 ACK。

“检查相机”只检测相机和预览是否可用，不执行门锁控制。

### 9.3 授权用户管理

- 授权用户列表支持上下滚动；
- 人脸模板只保存在大禹本机；
- 点击“删除”可移除指定用户；
- 删除后该用户不再具有人脸控制权限。

## 10. 多设备工作方式

每块设备必须拥有唯一 `deviceId`，并通过 `device_hello.capabilities` 声明能力。

大禹按能力路由，不要求用户在每次控制前手动选择设备：

| 操作 | 路由依据 |
| --- | --- |
| 开灯、关灯 | `light` 绑定 |
| 空调控制、红外学习 | `ac` / `ir_learning` 绑定 |
| 窗帘控制 | `curtain` 绑定；当前兼容回退到空调控制器 |
| 开锁、上锁 | `door` 绑定 |
| 消警 | 实际报警来源设备 |
| 温湿度显示 | `sensor` 能力设备自动上报 |

“设备状态”中的当前选中设备只用于查看状态以及选择协议/加密切换目标，不应该限制日常控制。

## 11. 通信与安全机制

### 11.1 两层保护

设备与大禹之间采用两层保护：

1. 传输层：MQTTS 或 HTTPS；
2. 业务层：SM4-GCM 或 AES-GCM。

即使启用了 TLS，业务数据仍会继续使用 GCM 加密。

### 11.2 本地服务端口

| 服务 | 明文端口 | TLS 端口 | 用途 |
| --- | ---: | ---: | --- |
| MQTT | 1883 | 8884 | 上报、命令、ACK、密钥协商 |
| HTTP | 3000 | 3444 | HTTP 上报和命令轮询 |

大禹应用内部通过 `127.0.0.1:1883` 连接本机 Broker。该明文回环流量不会离开大禹。

### 11.3 设备 Topic

```text
命令：dayu/cmd/{deviceId}
传感器：device/{deviceId}/sensor
空调/ACK：device/{deviceId}/ac
状态：device/{deviceId}/status
ECDH：key/ecdh/pub/{deviceId}
```

### 11.4 ACK 要求

设备 ACK 应包含：

- 真实 `deviceId`；
- 原命令的 `seq`；
- `cmd`；
- `ok`；
- 必要的状态字段。

大禹同时校验 `deviceId` 和 `seq`，避免多设备之间串包。

## 12. 常见问题处理

### 12.1 控制一直显示“同步中”

依次检查：

1. 设备是否在线；
2. 设备是否声明对应能力；
3. 设备绑定是否正确；
4. 设备与大禹的协议是否一致；
5. 加密算法和动态密钥是否一致；
6. Broker 是否收到命令；
7. 设备 ACK 是否带正确 `deviceId` 和 `seq`。

### 12.2 MQTT 能连接但控制无效

- 检查命令是否发到 `dayu/cmd/{deviceId}`；
- 检查是否错误使用公共 `dayu/cmd`；
- 检查设备是否订阅自己的唯一 Topic；
- 检查能力绑定是否指向正确设备。

### 12.3 HTTP 轮询正常但收不到命令

- 检查轮询参数是否使用真实 `deviceId`；
- 检查返回中的 `hasCommand`；
- 检查字段名是否为 `payloadBase64`；
- 检查命令是否进入对应设备的独立队列；
- 检查设备是否误用其他设备的动态密钥。

### 12.4 TLS 握手失败

- 确认大禹 IP 没有写反；
- MQTT TLS 端口应为 `8884`；
- HTTPS 端口应为 `3444`；
- 检查大禹和设备是否处于同一局域网；
- 确认证书已随配网配置正确安装；
- 如果设备已切到 HTTP，应停止无意义的 MQTT 自动重连。

### 12.5 红外学习卡住

- 确认当前绑定的是具有 `ir_learning` 能力的设备；
- 遥控器对准红外模块并靠近一些；
- 不要重复发送 `learn-all`；
- 不要同时发送单键 `learn`；
- 学习过程中不要切协议、切算法或重协商密钥；
- 单键最长可能等待 65 秒。

### 12.6 界面突然严重卡顿

如果触摸、弹窗和页面切换同时变慢，可能是 OpenHarmony 系统桌面进程异常占用 CPU，而不是禹家页面本身。

建议顺序：

1. 等待数秒确认是否恢复；
2. 关闭不需要的相机界面；
3. 退出并重新打开禹家终端；
4. 仍未恢复时重启大禹。

重启应用或大禹后，设备会重新发送 `device_hello` 并恢复注册。

### 12.7 USB 配网收不到 ACK

- 更换可靠的数据线或 USB 接口；
- 重新扫描 USB 设备；
- 重新执行“自动探测”；
- 设备确认已经解析配置时，可再次尝试“保存重启”；
- 避免把设备启动心跳文本误认为二进制 ACK。

## 13. 截图工具

项目根目录提供：

```text
screenshot.bat
```

在 Windows 电脑上双击后会：

1. 检查大禹 HDC 连接；

## 14. 部署与维护说明

- `deploy.bat` 用于构建、安装并启动禹家终端；
- 当前正式 Broker 位于大禹应用内部；
- 电脑端旧 Broker 脚本仅作为兼容和回退工具；
- 不要同时让设备连接电脑旧 Broker 和大禹 Broker；
- 更新应用不会删除设备红外模块 Flash 中已经学习的编码；
- 人脸和声纹模板保存在大禹本地，重新安装应用前应先确认是否需要备份。

## 15. 推荐日常使用流程

首次安装：

1. 为大禹设置固定局域网地址；
2. 打开禹家终端；
3. 使用 USB 逐块接入设备；
4. 确认设备发送 `device_hello`；
5. 检查设备能力和自动绑定；
6. 测试照明、空调、窗帘和门锁；
7. 完成空调红外学习；
8. 录入人脸和声纹；
9. 测试报警和消警 ACK；
10. 保持 MQTTS/HTTPS 和业务层加密开启。

日常使用：

1. 保持大禹和设备接入家庭 Wi-Fi；
2. 保持禹家终端运行；
3. 在总览页直接控制设备；
4. 使用“小禹小禹”唤醒语音助手；
5. 仅在新增设备或维护时进入设置页；
6. 设备异常时先检查设备状态、绑定、协议和 ACK。
