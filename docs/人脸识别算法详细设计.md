# 人脸识别算法详细设计文档

> 文档定位：本文仅描述当前工程中实际运行的人脸识别算法、数学过程、参数、边界条件与算法风险，不展开摄像头页面、用户管理界面、门锁联动等功能设计。
>
> 代码基线：`entry/src/main/cpp/face_inference.cpp`、`entry/src/main/ets/face/FaceAccessManager.ets`。

## 1. 算法目标与问题定义

当前系统实现的是一个本地、小规模、开放集（open-set）人脸识别系统。算法输入为摄像头采集的一张图像，输出包括：

- 一个 512 维单位人脸特征向量；
- 当前图像中的最大人脸检测置信度；
- 与本地已注册人员模板的最高余弦相似度；
- 当最高相似度达到阈值时给出的身份判定，否则判定为未知人员。

设本地模板库为

$$
\mathcal{G}=\{(u_i,\mathbf{t}_i)\}_{i=1}^{N},\quad \mathbf{t}_i\in\mathbb{R}^{512},\quad \|\mathbf{t}_i\|_2=1
$$

其中 $u_i$ 是用户身份，$\mathbf{t}_i$ 是其注册模板。对查询图像 $I$，特征网络生成查询向量 $\mathbf{q}$。系统求解：

$$
i^*=\arg\max_i \mathbf{q}^{\mathsf T}\mathbf{t}_i
$$

并按阈值 $\tau=0.75$ 做开放集决策：

$$
\hat{y}=\begin{cases}
u_{i^*}, & \mathbf{q}^{\mathsf T}\mathbf{t}_{i^*}\ge \tau\\
\text{unknown}, & \text{otherwise}
\end{cases}
$$

当前系统是 **1:N 识别**，不是只验证某一声明身份的 1:1 人脸验证。

## 2. 当前实际生效的算法链路

```text
JPEG 图像
  → 解码并缩放为 640×640 RGBA
  → SCRFD 人脸检测（仅取全图最高分候选）
  → 整幅图中心正方形裁剪
  → 最近邻采样缩放至 112×112
  → RGB 通道归一化：(x - 127.5) / 128
  → MobileFaceNet/ArcFace 系列识别模型 w600k_mbf
  → 512 维特征
  → L2 归一化
  → 与全部有效模板计算余弦相似度
  → Top-1 + 固定阈值 0.75 判决
```

需要特别说明：当前虽然执行了人脸检测并加载了 106 点关键点模型，但识别输入仍是**整图中心裁剪**。检测框裁剪、检测置信度硬门控和关键点对齐均未进入实际判决链路。

## 3. 模型组成

### 3.1 人脸检测模型

运行时模型为 `det_500m.ncnn.param/bin`，对应 SCRFD 轻量级检测器。输入尺寸固定为 $640\times640$，输入张量布局为三通道 RGB。

像素归一化公式为：

$$
x' = \frac{x-127.5}{128.0}
$$

检测器输出三组分类结果和三组边框回归结果，对应步长：

| 特征层 | 网格尺寸 | 步长 |
|---|---:|---:|
| P3 | $80\times80$ | 8 |
| P4 | $40\times40$ | 16 |
| P5 | $20\times20$ | 32 |

当前后处理不是完整的多目标检测流程，而是分别从三个尺度中寻找最大分类分数，再从三个最大值中选出全局最大值。设最大响应所在网格为 $(x,y)$，步长为 $s$，网络回归距离为 $(l,t,r,b)$，则原图坐标中的检测框为：

$$
\begin{aligned}
x_0 &= \frac{xs-ls}{\alpha_x}, & y_0 &= \frac{ys-ts}{\alpha_y}\\
x_1 &= \frac{xs+rs}{\alpha_x}, & y_1 &= \frac{ys+bs}{\alpha_y}
\end{aligned}
$$

其中 $\alpha_x=640/W$、$\alpha_y=640/H$。检测框随后以中心为基准扩展为正方形，边长为：

$$
L=1.35\max(x_1-x_0,\ y_1-y_0)
$$

并裁剪到图像边界内。

但是，在当前编译配置 `kEnableDetectedFaceCrop=false` 下，该检测框不会用于识别网络输入。检测最大分数仅作为 `qualityScore` 返回；它本质上是**人脸存在置信度**，并不是包含清晰度、姿态、遮挡、曝光的综合质量分。

### 3.2 人脸关键点模型

运行时加载 `2d106det.ncnn.param/bin`，即 106 点二维人脸关键点模型。当前加载成功是后端“就绪”的必要条件之一，但代码中没有执行该网络的 forward，也没有利用眼角、鼻尖、嘴角等关键点进行相似变换对齐。

因此，当前版本不能声称已实现 106 点人脸对齐。模型处于“资源和网络已加载、算法链路未接入”状态。

### 3.3 人脸特征模型

运行时模型为 `w600k_mbf.ncnn.param/bin`。从文件名和输入输出形式看，它是 InsightFace 模型包中的 MobileFaceNet 级别轻量识别模型，训练数据标识为 W600K；工程实际依赖的是转换后的 NCNN 图结构和权重。

模型输入：

$$
\mathbf{X}\in\mathbb{R}^{3\times112\times112}
$$

模型输出：

$$
\mathbf{z}\in\mathbb{R}^{512}
$$

输出经 L2 归一化：

$$
\mathbf{e}=\frac{\mathbf{z}}{\sqrt{\sum_{j=1}^{512}z_j^2}}
$$

只有输出维度严格等于 512 时，本次推理才被标记为成功。

工程资源中还保留 `1k3d68.onnx` 和 `genderage.onnx`，但当前 NCNN 主链路既不加载也不执行这两个模型，因此 3D 关键点、性别和年龄估计不属于当前识别算法。

## 4. 图像预处理

### 4.1 JPEG 解码

拍照输出首先取得 JPEG 数据，再通过系统图像解码器生成：

- 尺寸：$640\times640$；
- 像素格式：RGBA8888。

ArkTS 与 C++ 之间传递的二进制布局为：

```text
offset 0..3   : width，uint32，小端
offset 4..7   : height，uint32，小端
offset 8..end : RGBA 像素，每像素 4 字节
```

### 4.2 当前中心裁剪策略

对宽高为 $(W,H)$ 的输入，取：

$$
C=\min(W,H),\quad x_c=\frac{W-C}{2},\quad y_c=\frac{H-C}{2}
$$

得到居中的 $C\times C$ 正方形区域，再采样到 $112\times112$。当前上游已经请求解码成 $640\times640$，正常情况下中心裁剪实际上覆盖整张解码图。

采样通过整数坐标映射完成，等价于最近邻采样，而非双线性插值。对目标像素 $(x,y)$：

$$
x_s=x_c+\left\lfloor\frac{xC}{112}\right\rfloor,\quad
y_s=y_c+\left\lfloor\frac{yC}{112}\right\rfloor
$$

随后取 RGBA 中的 R、G、B 三通道，并执行相同的线性归一化：

$$
R'=(R-127.5)/128,
G'=(G-127.5)/128,
B'=(B-127.5)/128
$$

### 4.3 预处理版本

模板和查询均携带 `preprocessVersion="center"`。匹配时只比较预处理版本相同的特征，从而避免未来切换到检测框裁剪或关键点对齐后，将不同特征分布直接混用。

当前 `cropMode` 虽会随推理结果返回，但存储和过滤使用的是 `preprocessVersion`。由于检测框裁剪关闭，实际 `cropMode` 始终为 `center`。

## 5. 注册模板生成

一个用户可采集 $K$ 条样本特征：

$$
\mathbf{e}_1,\mathbf{e}_2,\ldots,\mathbf{e}_K
$$

系统先逐维计算算术平均，再进行一次 L2 归一化：

$$
\bar{\mathbf{e}}=\frac{1}{K}\sum_{k=1}^{K}\mathbf{e}_k,
\qquad
\mathbf{t}=\frac{\bar{\mathbf{e}}}{\|\bar{\mathbf{e}}\|_2}
$$

这相当于在单位超球面上用“归一化均值方向”作为身份原型。多样本平均能够抑制单帧噪声，但当前实现存在以下前置假设：

- 所有样本维度相同；
- 所有样本确实属于同一人；
- 样本质量近似，因而采用等权平均；
- 样本中没有无脸图、多人图或严重偏姿态图。

代码目前没有验证后续样本维度是否与第一条相同，也没有按检测置信度加权，更没有注册样本间一致性检查。因此低质量或错误样本会直接拉偏模板方向。

模板以 512 个浮点数的 JSON 数组持久化。读取时会重新进行 L2 归一化，以减小序列化或历史数据造成的模长偏差。

## 6. 相似度计算与身份判决

查询特征进入匹配前再次 L2 归一化。对查询 $\mathbf{q}$ 和模板 $\mathbf{t}_i$，代码计算点积：

$$
s_i=\sum_{j=1}^{512}q_jt_{ij}
$$

由于二者都是单位向量，上式等价于余弦相似度：

$$
s_i=\cos\theta_i\in[-1,1]
$$

匹配过程是线性遍历，跳过以下模板：

- 用户被禁用；
- 特征为空；
- `preprocessVersion` 与查询不一致。

取最高分 $s^*=\max_i s_i$。默认阈值为 0.75，且代码使用 `>=`：

- $s^*\ge0.75$：接受 Top-1 身份；
- $s^*<0.75$：拒绝为未知人员。

即使最终拒绝，结果结构仍保留最高分候选的用户 ID 和姓名，调用方应以 `matched` 字段作为授权依据，不能只检查候选是否非空。

### 6.1 时间与空间复杂度

设模板数为 $N$、特征维数为 $D=512$：

- 单次穷举匹配时间复杂度：$O(ND)$；
- 模板内存复杂度：$O(ND)$；
- 单模板纯 float32 特征约占 $512\times4=2048$ 字节，不含 JSON 文本膨胀和元数据。

家庭场景中 $N$ 通常很小，线性扫描优于引入近似最近邻索引；当模板达到数千以上时才需要考虑向量索引、分片或矩阵化批量点积。

## 7. 检测后处理的当前特性

当前 SCRFD 后处理与标准通用检测实现有明显差异：

1. 只选择全局最高分位置，不生成候选集合；
2. 没有非极大值抑制（NMS）；
3. 没有多脸计数和主脸选择策略；
4. 分类输出直接取最大数值，代码中没有显式 sigmoid；是否已经在转换图内包含激活需以 NCNN param 为准；
5. 边框解码中的 anchor 索引未参与实际坐标计算。

对于“只需要一张主脸”的设备场景，第 1、2 点可以降低后处理开销。但因为检测框当前不参与裁剪，这套后处理对最终 embedding 的直接影响仅限于返回 `qualityScore`，并不阻止无脸图继续进入识别网络。

## 8. 算法开关与真实行为

| 编译常量 | 当前值 | 当前行为 |
|---|---:|---|
| `kEnableNcnnForward` | `true` | 执行真实 NCNN 检测与特征推理 |
| `kEnableNcnnZeroForwardProbe` | `false` | 不使用全零输入探测 |
| `kEnableFacePresenceHardGate` | `false` | 检测失败或分数低于 0.65 时不会硬拒绝 |
| `kEnableDetectedFaceCrop` | `false` | 不采用检测框，始终使用中心裁剪 |

代码中已经预留两个阈值：

- 检测硬门控阈值：0.65；
- 启用检测框裁剪时的最低检测分：0.45。

二者在当前配置下都不会改变最终身份判决。

## 9. 数值稳定性与异常边界

### 9.1 零向量

当向量 L2 范数不大于 $10^{-6}$ 时，归一化函数返回原向量副本。若它是零向量，与任意正常模板的点积为 0，通常无法通过 0.75 阈值。但系统没有显式把零向量标为非法。

### 9.2 维度不一致

余弦计算发现两向量为空或维度不同时返回 -1。识别网络输出必须为 512 维才返回成功；但注册时的多样本平均没有逐条维度校验，异常短向量可能产生 `undefined/NaN` 污染。

### 9.3 非有限数

持久化数据读取只检查元素类型为 `number`，未显式排除 `NaN` 和无穷值。标准 JSON 不能原生表示这些值，因此正常文件路径风险较低，但内存调用仍应增加 `Number.isFinite` 校验。

### 9.4 图像尺寸

Native 输入要求宽高至少为 16，且缓冲长度不少于 $8+4WH$。不满足时直接拒绝推理。

## 10. 当前算法能力边界

当前版本已经完成：

- 本地 NCNN 模型加载与真实 forward；
- SCRFD 单主脸置信度及边框解码；
- 512 维人脸特征生成和归一化；
- 多样本均值模板；
- 1:N 余弦相似度检索与开放集阈值拒绝；
- 预处理版本隔离。

当前版本尚未形成完整算法闭环的部分：

- **检测硬门控未启用**：无清晰人脸时仍可能生成 embedding；
- **检测框裁剪未启用**：人脸偏离画面中心时，特征会显著退化；
- **关键点对齐未实现**：旋转、侧脸、俯仰变化会直接进入识别模型；
- **活体检测未实现**：无法抵御照片、屏幕回放、视频或面具攻击；
- **综合质量评估未实现**：没有模糊、曝光、遮挡、姿态、脸尺寸判定；
- **多人场景未处理**：没有“拒绝多人”或指定主脸策略；
- **阈值未经项目数据标定**：0.75 是代码默认值，不等同于经过本设备数据验证后的安全阈值；
- **无 Top-1/Top-2 间隔约束**：两个相近身份分数非常接近时仍可能接受最高者；
- **模板未加密**：这属于数据安全问题，也会影响生物特征系统的整体风险等级。

因此，当前算法可描述为“已打通真实特征推理和本地比对的工程版本”，不应描述为已具备金融级、门禁级防伪能力。

## 11. 阈值标定方法

固定阈值必须基于目标摄像头、部署距离、光照和真实人员数据标定。建议构建两类分数：

- 同人分数集 $S_G$：同一人的不同采集之间的相似度；
- 异人分数集 $S_I$：不同人员采集之间的相似度。

对候选阈值 $\tau$ 计算：

$$
\mathrm{FAR}(\tau)=\frac{|\{s\in S_I:s\ge\tau\}|}{|S_I|}
$$

$$
\mathrm{FRR}(\tau)=\frac{|\{s\in S_G:s<\tau\}|}{|S_G|}
$$

其中 FAR 是误接受率，FRR 是误拒绝率。智能门锁属于安全敏感场景，应先规定可接受的 FAR 上限，再在满足该上限的阈值中选择 FRR 较低者，而不是只追求等错误率（EER）。

测试数据至少应覆盖：

- 白天、夜间、背光、室内暖光；
- 正脸、左右偏转、俯仰、不同距离；
- 眼镜、口罩、帽子、发型变化；
- 设备长期使用后的跨日、跨周采集；
- 家庭成员之间较相似的困难负样本；
- 无脸、多人和屏幕照片等非正常输入。

需要分别记录检测召回率、有效样本通过率、FAR、FRR、识别耗时及分位数，不能仅统计总体“识别准确率”。

## 12. 推荐的算法演进顺序

### P0：让检测真正约束识别

1. 开启人脸存在硬门控；
2. 启用检测框裁剪；
3. 对无脸和多人输入明确拒绝；
4. 将预处理版本升级为新值，避免与旧的 `center` 模板混用；
5. 要求旧用户重新录入，或实施受控模板迁移。

### P1：接入关键点对齐

执行关键点网络，从 106 点中取得稳定的五点子集，估计到标准 112×112 模板的相似变换：

$$
\mathbf{p}'=s\mathbf{R}\mathbf{p}+\mathbf{t}
$$

使用最小二乘估计尺度 $s$、旋转矩阵 $\mathbf{R}$ 和平移 $\mathbf{t}$，再做仿射采样。对齐后需重新标定识别阈值并升级预处理版本。

### P2：建立质量控制

在注册和识别前加入：

- 人脸尺寸占比；
- 模糊度（如 Laplacian 方差或训练式质量模型）；
- 曝光与动态范围；
- yaw/pitch/roll；
- 遮挡和关键点完整性；
- 多帧稳定性。

注册模板可改为质量加权均值：

$$
\mathbf{t}=\operatorname{norm}\left(\frac{\sum_k w_k\mathbf{e}_k}{\sum_k w_k}\right)
$$

### P3：强化判决

在绝对阈值外增加最佳与次佳模板的分差约束：

$$
s_1\ge\tau \quad\land\quad s_1-s_2\ge\delta
$$

对于门锁授权，建议再叠加连续多帧一致性，而不是以单张照片直接形成最终安全决策。

### P4：活体与抗攻击

根据硬件条件引入 RGB 被动活体、动作挑战、红外或深度活体。单目 RGB 活体只能提高攻击成本，不能等价于具备深度/红外传感器的强活体能力。活体分数应作为授权硬门控，而不是简单加到人脸相似度上。

## 13. 算法伪代码

### 13.1 当前查询流程

```text
function recognize(jpeg, gallery, threshold = 0.75):
    rgba = decode_and_resize(jpeg, 640, 640)

    detection = scrfd(rgba)
    qualityScore = max_detection_score(detection)
    # 当前不因检测失败而退出，也不使用检测框

    crop = center_square_crop(rgba)
    input = nearest_resize(crop, 112, 112)
    input = (RGB(input) - 127.5) / 128.0

    embedding = recognizer(input)
    if dimension(embedding) != 512:
        return inference_error
    query = l2_normalize(embedding)

    bestScore = -1
    bestUser = none
    for user in gallery:
        if not user.enabled:
            continue
        if user.preprocessVersion != "center":
            continue
        score = dot(query, user.embedding)
        if score > bestScore:
            bestScore = score
            bestUser = user

    matched = bestUser != none and bestScore >= threshold
    return matched, bestUser, bestScore, qualityScore
```

### 13.2 当前注册流程

```text
function enroll(name, sampleEmbeddings):
    dimension = length(sampleEmbeddings[0])
    mean = zeros(dimension)
    for embedding in sampleEmbeddings:
        mean += embedding
    mean /= count(sampleEmbeddings)
    template = l2_normalize(mean)
    save(name, template, preprocessVersion="center")
```

## 14. 结论

当前人脸识别核心是“SCRFD 检测信息 + MobileFaceNet 级 512 维表征 + 单原型余弦最近邻”。特征推理和模板匹配已经是真实可运行实现，但算法链路仍以中心裁剪为主，检测、关键点和质量信息尚未真正约束身份判决。

从算法可靠性看，下一阶段最重要的不是继续增加模型文件，而是把已经具备的检测框和关键点能力接入预处理闭环，建立质量门控，并使用本设备真实数据重新标定阈值。完成这些工作后，系统才能从“可运行的人脸相似度识别”演进为“可评估、可控风险的人脸认证算法”。
